# Change: 图层精修——自动抠图分层与固定画布拼接

## Why
对一张已满意的指定尺寸宣发图做精修时，用户常常想把画面里的角色/物件/LOGO/文案**各自拆成可独立摆放的图层**，再挪位置、缩放、调层叠——而不是整图再生一次。这件事应当是**对那张图本身的就地精修**：入口就在该图上（右键），画布尺寸**锁定为原图尺寸不可调**，分层由系统**自动分析 + 大模型抠图**完成，而非让用户一个个手动框选。gpt-image-2 做不出透明底（现有 spec 已把它的「透明底」降级改写），所以抠图层必须走**支持透明底的 Gemini 适配器**。

## What Changes
- 新增 **前景主体检测**：对源图调用视觉模型，列出可独立分层的前景主体（角色/物件/LOGO/文案块），各带简短中文描述与归一化包围盒；检测不到可分层主体时明确反馈。
- 新增 **大模型抠图意图 `extract_layer`**：把某个主体抠成**源图同尺寸的透明底 PNG 图层**，强制走支持透明底供应商（Gemini），未配置时明确不可用、不静默产出不透明图。复用既有异步/进度/主备切换/注入防护/低发散骨架/重试。
- 新增 **背景补全意图 `fill_background`**：移除前景主体、补全为干净完整背景，作为**锁定的底层**。输出不透明，优先 Gemini、可降级到默认适配器。
- 新增 **图层精修编排端点 `layer-split`**：检测主体 → 并发驱动「每主体一次 `extract_layer` + 一次 `fill_background`」→ 汇总返回**有序图层**（背景在最底，主体按重要性排序），全部为源图尺寸。
- 新增/调整 **固定画布拼接**：入口在**该图右键 / 预览内「图层精修」**；画布尺寸 = **原图尺寸，锁定不可调**；底层为背景层（**锁定**：不可移动/缩放/移除/下移到其下）；主体层支持平移、等比缩放、层叠、移除（非破坏式）。
- **合成导出**：浏览器拍平为 PNG，经直连端点落库为新资产；时间轴归类为【拼接】。

非目标（本期不做）：旋转、滤镜、混合模式、羽化、多画布、手动逐主体框选（已由自动检测取代）。

## Impact
- 受影响 specs（新增能力）：`layer-compositing`（自动分层编排 + 固定画布拼接 + 合成导出 + 持久化端点）、`marketing-analysis`（前景主体检测）
- 受影响 specs（修改）：`image-generation`（`extract_layer` 抠图意图 + `fill_background` 背景补全意图 + 透明底供应商路由/降级）、`frontend-experience`（图层精修入口与固定画布交互）、`asset-workspace`（时间轴【拼接】事件）
- 复用既有能力（无需改契约）：`image-generation` 多供应商适配器 / 低发散骨架 / region_desc 注入防护 / 已生成图重试、`image-optimization` PNG 无损优化（保留 alpha）、`download-packaging`（透明 PNG 走兜底目录）、`asset-workspace` 资产编号/移除/会话隔离
- 受影响代码：`internal/vision`（多主体检测）、`internal/generation`（两个新意图 + 路由）、新增 `internal/layering`（编排端点）、新增 `internal/composite`（合成持久化端点）、前端预览/工作区（右键入口 + 固定画布编辑器 + 拍平导出）
