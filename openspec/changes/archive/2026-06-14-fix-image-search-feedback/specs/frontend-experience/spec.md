## MODIFIED Requirements

### Requirement: 长任务即时占位反馈
当系统创建长任务（生图、二次调整、生视频、**找图等**）时，系统 SHALL 在任务创建的瞬间经对话通道广播任务创建通知，前端据此**立即**于工作区「进行中」分组插入占位骨架卡片，并即时订阅该任务的进度流，使工作区在本轮对话尚未结束前即给出可见的 loading 反馈，而非等到对话回答完成才刷新。该占位机制 SHALL 独立于 Agent 的工具回调时序，确保任务一经创建占位即出现。对于一次产出多张资产的找图任务，任务创建通知 SHALL 携带本次请求的图片张数，前端 SHALL 据此**按张数占位**（占位数与请求张数一致），并在每张资产到达时就地回填对应占位。

#### Scenario: 二次调整即时出现占位
- **WHEN** 用户在工作区点击某张图发起二次调整，且系统据此创建了生图任务
- **THEN** 工作区在任务创建后随即在「进行中」分组出现一张占位骨架卡片并展示阶段化 loading 进度
- **AND** 该占位在本轮对话回答结束之前即已出现，无需等待对话 done

#### Scenario: 占位随进度演进至完成
- **WHEN** 该任务的进度事件持续到达
- **THEN** 占位骨架按阶段更新进度，任务完成后替换为最终产物卡片
- **AND** 同一任务 id 在工作区不重复出现多张卡片

#### Scenario: 占位不依赖工具回调时序
- **WHEN** Agent 的工具完成回调延迟、缺失或时序不确定
- **THEN** 占位骨架仍在任务创建时即出现，不受该回调影响

#### Scenario: 找图按张数占位并逐张回填
- **WHEN** 用户发起一次找 N 张图的找图任务
- **THEN** 工作区在任务创建后随即出现 N 个占位
- **AND** 每张图片下载完成时就地回填其中一个占位，无需手动刷新
