# Polish Studio UI（UI 体验第一次优化）

## Why
当前前端（`web/static` 三文件：index.html / styles.css / app.js）已能跑通对话→生图→裁剪→打包全流程，但作为对外展示的「成熟 Agent 产品」观感仍有差距：

- 缺少科技感动效（SVG 动画、按钮/细节过渡），整体偏静态。
- 工作区资产仅能左键多选，缺少「直接查看单图」的轻量入口；卡片单列堆叠，无法拖拽调序，浏览效率低。
- 「规划中的能力」路线图占据右栏主视觉过重，喧宾夺主。
- 各投放渠道（27 个）缺少可识别的视觉标识，选择尺寸时辨识成本高。
- 用户输入多为口语化描述，直接喂给生图模型质量不稳定，缺少「一键优化提示词」入口。

本提案在**保持现有原生前端栈**（不引入 Vite/框架，零构建步骤、Go embed 不变）的前提下，做一轮以设计师视角主导的体验打磨，并新增一个轻量后端「提示词优化」端点支撑口语化输入改写。

## What Changes
- **设计语言与动效（前端）**：抽出统一设计 token，补充品牌区 SVG 科技感动画、按钮/卡片/弹层的微交互过渡，提升关键路径流畅度（滚动锚定、加载过渡），不改变信息架构。
- **渠道视觉标识（前端）**：为 27 个渠道提供「首字母/品牌色色块」占位标识，在尺寸选择器渠道列表与已选项处展示，降低辨识成本。**不抓取第三方真实商标**（版权/来源不稳定）。
- **单图直接查看入口（前端）**：在每个资产卡上新增一个独立的「查看」小按钮，点击直接打开放大预览（lightbox），与「左键多选」解耦。
- **工作区拖拽排序 + 多列分栏（前端）**：已完成资产支持拖拽调整展示顺序（前端展示态，不持久化到后端），并以多列网格/瀑布流分栏呈现。
- **路线图视觉降权（前端）**：将「规划中的能力」从右栏常驻大卡片改为收敛式呈现（折叠/紧凑入口），释放工作区主视觉。
- **提示词优化（前后端）**：输入框新增「优化提示词」小按钮；新增后端 `POST /api/session/{id}/prompt/optimize` 端点，用会话理解模型把口语化输入改写为结构化生图提示词，返回文本回填输入框供用户确认后再发送（不直接触发生图）。

**保持不变**：前端技术栈（原生 HTML/CSS/ESM、Go embed）、对话/生图/裁剪/视频/爬取的后端能力边界、白名单意图集合。

## Impact
- Affected specs:
  - `frontend-experience`（MODIFIED：品牌化科技感界面、交互流畅度、未来功能路线图展示；ADDED：渠道视觉标识、单图直接查看入口、工作区拖拽排序与分栏、提示词优化入口）
  - `conversation-orchestration`（ADDED：提示词优化端点）
- Affected code:
  - `web/static/index.html`、`web/static/styles.css`、`web/static/app.js`（重构为更清晰的模块/分区，仍为原生栈）
  - 新增渠道标识所需的渠道→品牌色映射（前端常量或读取 `configs/channels.json` 既有字段）
  - `cmd/server/main.go`、`internal/agent`（新增 prompt-optimize 编排与路由）
- 非目标（Out of scope）：升级前端到 Vite/框架；抓取/下载第三方渠道真实 logo；把工作区排序持久化到后端存储；改动生图/裁剪算法本身。
