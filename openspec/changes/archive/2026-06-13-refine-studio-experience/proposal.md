# Change: 体验第一次优化专项

## Why
MVP 已跑通对话生图/裁剪/下载主链路，但首轮真实试用暴露了一批体验与正确性问题：布局重心放错（工作区比对话还宽）、选中的图与实际操作的图错位、思考过程不可见、长任务等待无反馈、切尺寸与打包能力不完整。这一批问题集中在"看起来像个成熟 Agent 产品"和"操作结果可信"两条线上，需要一次成体系的体验优化把它们一起收敛，而不是零散补丁。

## What Changes
- **品牌与科技感（需求0）**：强化首屏品牌与服务意识，统一精简科技感视觉语言。
- **思考过程真·逐字流式 + 折叠（需求1）**：重写模型层为 provider SSE 真流式，思考内容逐字涌现，回答产出后思考块自动折叠、可展开回看。**BREAKING**（chatmodel 传输层由"先 Generate 再切块"改为真流式）。
- **交互流畅度优化（需求2）**：消除阻塞感，补齐过渡/骨架/滚动锚定等流畅度细节。
- **操作目标一致性修复（需求3）**：选中第 N 张却操作了第一张的错位 bug，确保右键/选中/二次调整严格作用于目标资产。
- **Cursor 式对话区与状态严控（需求4）**：对话框为核心，工具调用以受控的结构化卡片呈现（pending/运行/成功/失败/结果摘要），当前状态严格可见。
- **用户偏好关键词记录（需求5）**：前端从用户消息提取高频关键词，写入偏好角落；无任何记录时角落隐藏。
- **未来功能路线图展示（需求6）**：在前端以"规划中"卡片展示自学习、团队知识库、生视频等 Agent 建设方向（仅展示，不实现功能本身）。
- **长任务等待体验（需求7）**：生图/生视频耗时场景对齐头部产品做法（进度阶段、预计、占位反馈、可继续操作）。
- **异常小弹窗（需求8）**：沿用既有 toast 规范（已满足，纳入验收基线，不新增行为）。
- **切尺寸全选 + zip 分目录（需求9）**：尺寸选择器支持"全选当前渠道可裁剪尺寸"；批量打包按渠道/尺寸分目录组织 zip。
- **工作区批量切尺寸（需求10）**：工作区多选资产后，除打包下载外新增"批量切尺寸"入口。
- **布局比例 3:7（需求11）**：工作区:对话窗口 = 3:7，把视觉与操作重心交还对话区。
- **二次调整产物尺寸继承源图（需求12）**：基于某张图修改/调整时，产物尺寸与该源图原尺寸一致。

## Impact
- Affected specs: `frontend-experience`、`conversation-orchestration`、`asset-workspace`、`download-packaging`、`image-generation`
- Affected code:
  - 后端: `internal/agent/chatmodel.go`（真流式）、`internal/agent/agent.go`（reasoning 增量透传）、`internal/generation/service.go` + `provider.go`（尺寸继承）、`internal/download`（分目录 zip）、`internal/crop`（批量入口复用）
  - 前端: `web/static/app.js`、`web/static/index.html`、`web/static/styles.css`（布局比例、思考折叠、工具状态卡、偏好角落、路线图卡、长任务反馈、目标一致性、选择器全选、批量切尺寸）
