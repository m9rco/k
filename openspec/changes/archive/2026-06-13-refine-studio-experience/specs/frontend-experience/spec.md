## MODIFIED Requirements

### Requirement: 品牌化科技感界面
系统 SHALL 提供精简、强科技感的品牌化界面，传达明确的产品与服务意识，并在全站使用统一的视觉语言（配色、留白、图标、动效节奏）以建立成熟 Agent 产品的观感。

#### Scenario: 首屏品牌呈现
- **WHEN** 用户进入应用
- **THEN** 界面以精简科技感视觉呈现品牌与核心能力入口
- **AND** 顶部明确传达"宣发素材助手"的产品与服务定位

#### Scenario: 统一视觉语言
- **WHEN** 用户在对话区、工作区、弹层、通知之间切换
- **THEN** 各区域沿用一致的配色、圆角、间距与过渡节奏，无割裂感

### Requirement: Agent 式对话与工具调用呈现
系统 SHALL 提供参考主流 Agent 产品（如 Cursor）的对话区，将对话区作为产品核心区域；工具调用 SHALL 以结构化卡片呈现，并严格受控地展示每次调用的生命周期状态（待执行/执行中/成功/失败）与结果摘要，同时清晰呈现 context 状态与当前会话信息。

#### Scenario: 展示工具调用生命周期
- **WHEN** Agent 调用某个工具（生图/裁剪/下载等）
- **THEN** 对话区以结构化卡片展示该工具名与参数摘要
- **AND** 卡片随执行进展严格更新为执行中→成功/失败，并在结束时展示结果摘要或错误
- **AND** 任一工具调用不会停留在无结局的"执行中"状态

#### Scenario: Context 状态面板
- **WHEN** 会话进行中
- **THEN** 界面展示当前 context 使用状态与当前会话信息

### Requirement: 偏好角落占位
系统 SHALL 预留一个偏好展示角落，用于呈现从用户对话中提取的高频关键词偏好；在没有任何可展示偏好/操作记录时该角落不展示。

#### Scenario: 无偏好时隐藏
- **WHEN** 用户尚无任何可分析的偏好记录
- **THEN** 偏好角落不展示

#### Scenario: 提取并展示关键词偏好
- **WHEN** 用户发送了若干条包含可提取关键词的消息
- **THEN** 前端从消息中提取高频关键词（过滤停用词）并在偏好角落展示
- **AND** 关键词随后续对话累计更新

### Requirement: 分层尺寸选择器
前端 SHALL 以**分层选择器**呈现尺寸目录，替代原先的"平台分组一行平铺胶囊"方式，以承载 23+ 渠道、上百条尺寸的规模。选择器 SHALL 支持：先按渠道筛选/搜索 → 在选中渠道内按素材类型分组展示尺寸胶囊 → 跨渠道累加多选 → 一次性确认批量裁剪。选择器 SHALL 提供"全选当前渠道可裁剪尺寸"的快捷操作。

#### Scenario: 渠道筛选与搜索
- **WHEN** 用户打开尺寸选择器
- **THEN** 前端展示按速查分组（如外渠/手机厂商/腾讯内渠/PC）组织的渠道列表
- **AND** 提供渠道搜索框以快速定位渠道

#### Scenario: 渠道内按素材类型展示
- **WHEN** 用户选中某个渠道
- **THEN** 前端按素材类型（截图/ICON/视频封面/推广图/资源位/H5 等）分组展示该渠道的尺寸胶囊
- **AND** 每个胶囊展示尺寸名、宽×高

#### Scenario: 跨渠道多选与批量确认
- **WHEN** 用户在不同渠道下分别勾选若干尺寸
- **THEN** 前端将所选项聚合到"已选 N 项"区域并保持跨渠道累加
- **AND** 用户确认后以所选尺寸的唯一 id 列表发起一次批量裁剪

#### Scenario: 全选当前渠道可裁剪尺寸
- **WHEN** 用户在某渠道下点击"全选"
- **THEN** 前端将该渠道下全部**可裁剪**尺寸加入已选集合
- **AND** 不可裁剪（producible=false）的尺寸不被选中
- **AND** 已选计数与跨渠道累加保持一致

## ADDED Requirements

### Requirement: 思考过程流式展示与折叠
系统 SHALL 将 Agent 的思考过程以**逐字流式**方式实时展示在对话区，作为与最终回答视觉区分的"思考块"；当本轮回答开始产出或进入工具调用时，思考块 SHALL 自动折叠为收起态，并保留可展开入口供用户回看。

#### Scenario: 思考逐字涌现
- **WHEN** Agent 在产出回答前进行思考且模型返回思考内容
- **THEN** 前端以逐字增量方式实时渲染思考文本于独立的思考块
- **AND** 思考块在视觉上与最终回答明显区分

#### Scenario: 产出结论后折叠
- **WHEN** 本轮回答开始产出或工具调用开始
- **THEN** 对应思考块自动折叠为收起态
- **AND** 用户可点击展开回看完整思考内容

#### Scenario: 无思考内容时不展示
- **WHEN** 模型未返回任何思考内容
- **THEN** 前端不展示空的思考块

### Requirement: 主次布局比例
前端 SHALL 以工作区:对话窗口 = 3:7 的比例组织主布局，使对话区成为视觉与操作的主区域；在窄屏下 SHALL 退化为上下堆叠的响应式布局。

#### Scenario: 宽屏 3:7 布局
- **WHEN** 用户在宽屏访问应用
- **THEN** 对话窗口占据约 70% 宽度、工作区约 30%

#### Scenario: 窄屏响应式堆叠
- **WHEN** 视口宽度低于响应式断点
- **THEN** 对话区与工作区改为上下堆叠排列且均可用

### Requirement: 交互流畅度
系统 SHALL 在关键交互路径上消除阻塞感：列表/资产更新使用过渡或骨架占位、流式内容追加时保持滚动锚定、操作反馈即时，避免出现长时间空白或界面跳动。

#### Scenario: 流式滚动锚定
- **WHEN** 对话区有流式内容持续追加
- **THEN** 视图保持锚定在最新内容，不产生抖动或错位

#### Scenario: 加载过渡
- **WHEN** 工作区资产或任务状态更新
- **THEN** 以过渡/骨架平滑呈现而非空白闪烁

### Requirement: 未来功能路线图展示
前端 SHALL 在界面中以"规划中"卡片展示 Agent 建设方向（如自学习偏好、团队知识库、生视频/动效等），明确标注为规划中能力；此处 SHALL 仅做展示，不实现对应功能本身。

#### Scenario: 展示规划能力
- **WHEN** 用户查看路线图区域
- **THEN** 前端以卡片列出规划中的能力及简述
- **AND** 每项明确标注"规划中"，不提供会触发后端功能的真实操作
