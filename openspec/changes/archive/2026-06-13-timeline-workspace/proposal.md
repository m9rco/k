# Timeline Workspace

## Why

当前工作区(`web/src/components/workspace/workspace-panel.tsx`)以**里程碑式三段**组织条目:进行中 / 已完成 / 失败,每段内是 `auto-fill` 多列网格,已完成资产支持自由拖拽重排(`orderedAssets` + `state.order`)。这种组织有两个体验问题:

1. **丢失了"创作过程"的叙事**:用户在一次会话里是按操作推进的——上传一张图 → 换背景 → 切尺寸 → 生视频。当前网格把所有产物平铺,完成顺序、谁从谁派生、什么时候做的,全都看不出来。Game Asset Studio 想传达的是"游戏宣发资产**工坊**"——一个能看见加工流水、看见素材如何被一步步打磨出来的地方,而平铺网格更像一个"图库",缺少工坊的过程感。
2. **任务与资产割裂**:进行中/失败任务单独成段(`task-card.tsx`),完成后才"跳"到已完成网格(`asset-card.tsx`),用户看不到"这张正在生成的图,是接在我刚才那次换背景之后的"这种连续关系。

审计还发现:后端 `AssetRecord.CreatedAt`(`internal/store/store.go`)已落库,但 `AssetView`(`internal/workspace/workspace.go`)**未把它序列化**给前端,前端 `Asset` 类型(`web/src/lib/types.ts`)也无时间字段——所以前端目前无法按真实时间排列。

## What Changes

为工作区**新增一种时间轴(生产流水)展示视图**,与既有的全部展示网格视图并存,默认仍为网格视图,用户可通过工作区的视图切换在二者间切换,营造"资产工坊"的加工过程感:

- **视图切换(网格默认 / 时间轴可选)**:工作区头部提供视图切换入口;默认网格视图(全部展示,按状态分段的多列网格),用户可切到时间轴视图;选择在会话内保持(sessionStorage)。两视图共享同一份数据与同一套编号/选择/批量/单图操作。
- **时间轴视图:事件串联**:每个节点是一次**创作事件**(上传、生图/编辑、切尺寸一对多产出、生视频、爬取),按发生先后沿主干线串联,直观呈现"流水线"。
- **时间轴视图:任务入轴**:进行中/失败任务作为时间轴**活跃端节点**就地呈现,完成后**原地**变为产物节点。(网格视图仍按状态分段。)
- **后端下发真实时间**:`AssetView` 序列化 `createdAt`;前端 `Asset` 增加 `createdAt`。节点按真实创建时间排序,并展示相对时间(如"3 分钟前")。
- **节点显示 LLM 对该操作的理解**:节点标题优先展示 agent 对本次操作的理解(由工具调用参数派生,如"换背景 · 黄昏的海边"),让用户看见每一步"LLM 怎么理解的"。
- **派生关系可见**:利用已有的 `parentId`,在节点上标注它由哪个素材加工而来(如"由 图2 加工"),强化"加工链路"。
- **工坊感视觉**:沿用既有沉静低装饰设计 token,用时间轴的节点圆点、主干线、动作短语等克制元素强化"工坊流水"氛围,不引入花哨色块。
- **编号保留,作为 LLM 锚点**:资产编号(图片"图N"、视频"视频N")继续存在,其唯一作用是让 LLM 在对话中识别用户指代的是哪一个 asset id;两视图角标一致。
- **取舍**:**移除已完成资产的自由拖拽重排**;编号/顺序统一按创建先后,稳定且与 LLM 沟通一致。保留多选、批量操作、单图操作。一次操作多产物的稳定聚合所需的"批次 id"留作后续后端工作(本提案用前端启发式聚合,见 design)。

本提案**不改动**:生图/裁剪/视频/爬取的底层逻辑、对话区、capsule、任务进度 SSE 协议、模型策略。

## Capabilities & Specs

- `asset-workspace`(MODIFIED/ADDED):里程碑分组 → 时间轴事件串联;任务入轴;资产创建时间下发;编号继续作为 LLM 锚点(图N/视频N),映射改为反映时间序(拖拽移除)。
- `frontend-experience`(MODIFIED/ADDED):里程碑分组与拖拽分栏的呈现方式改为时间轴;新增时间轴视觉(节点/连线/相对时间/派生标注/工坊感)。

## Impact

- 后端:`internal/workspace/workspace.go`(AssetView 加 createdAt 序列化)。无 schema 变更(created_at 已存在)。
- 前端:`web/src/lib/types.ts`(Asset.createdAt)、`web/src/components/workspace/workspace-panel.tsx`(分组→时间轴)、`asset-card.tsx` / `task-card.tsx`(节点化呈现 + 时间/派生标注)、`web/src/store/controller.ts`(排序按时间、移除拖拽态;编号映射随之按时间序)、可能新增 `timeline.tsx` / `timeline-node.tsx`。
- 协议:`createdAt` 为加法式字段,向后兼容。
- 风险:移除自由拖拽是行为变更(需在 spec 明确并保留多选/批量)。编号机制不变(仍是 LLM 锚点),仅因拖拽移除而映射顺序变为时间序——时间序天然稳定,对"图N→asset_id"沟通反而更一致。
