# image-generation Specification (delta)

## ADDED Requirements

### Requirement: 平台适配生图意图
系统 SHALL 提供 `adapt_platform` 生图意图，基于一张源图重绘出适配目标平台尺寸的素材，并由服务端提示词模板显式表达平台适配语义，使所有图生图模型（OpenAI 兼容与 Gemini 形态）都能准确理解目标。该意图 SHALL 复用既有颜色适配、参考图复用、注入防护、多供应商失效切换与异步管线。

模板 MUST 覆盖以下语义，全部由服务端组装、用户文本仅作为 sanitized slot 片段：
- 保留源图主体/角色与核心宣发意图；
- 为目标 `{orientation} {W}×{H} {渠道名} {素材类型名}` 广告位重新组织构图；
- 在新比例下**补全/延展画面而非裁掉主体**；
- 透传该尺寸的语义约束（如「无文案」「仅 logo」「圆角」「透明底」「安全区」）作为生图约束；
- 颜色与风格协调（复用既有 palette + 协调约束）。

#### Scenario: 按平台适配重绘并保留主体
- **WHEN** 系统以 `adapt_platform` 意图对源图发起生图，目标为某渠道某尺寸
- **THEN** 产物在目标尺寸下保留源图主体与核心宣发意图，画面被补全而非裁切
- **AND** 产物记录来源供应商并回填工作区

#### Scenario: 平台语义注入提示词
- **WHEN** 目标尺寸带有语义约束（如「无文案」「仅 logo」）
- **THEN** 系统将其作为约束注入最终生图提示
- **AND** 提示由服务端模板组装，模型无关地表达「保留意图 + 适配该平台尺寸 + 补全不裁切」

#### Scenario: 适配提示注入防护
- **WHEN** 用户在适配描述中包含试图改写系统指令的内容（如 "ignore previous instructions"）
- **THEN** 系统通过结构化 slot 承接用户输入并由服务端模板组装最终提示，剥离/忽略其中的控制类指令
- **AND** 用户文本不被直接拼接为可改写系统行为的提示

#### Scenario: 跨供应商一致
- **WHEN** 图生图后端在 `gpt-image-2` 与 Gemini 图像模型之间切换
- **THEN** `adapt_platform` 适配语义在两类适配器上均被模型正确理解并产出
- **AND** 主备失效切换、产物来源记录语义不变
