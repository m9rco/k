# Proposal: guide-ref-coverage

## 背景与问题

集邮册（stamp-album）的参考图行目前是一个**被动的上传图池**：用户把图片丢进去，系统不做任何引导，也不告诉用户"上传的这组图是否足够覆盖所有平台尺寸"。

从 `configs/channels.json` 的 139 个可产出尺寸来看，平台尺寸覆盖分布为：

| 比例族 | 尺寸数 | 代表 | 说明 |
|---|---|---|---|
| **竖版** (ratio < 0.80) | 20 个 | 9:16 截图、竖版广告 | 需要竖版参考图 |
| **方形/近方** (0.80–1.25) | 37 个 | 1:1 ICON、3:4 | 需要方形参考图 |
| **横版 16:9** (1.25–1.90) | 59 个 | 16:9 截图、封面、Banner | 需要横版参考图 |
| 横版宽 (1.90–2.60) | 13 个 | ~2:1 宽横图 | 横版覆盖可兼顾 |
| **超宽 >2.60** | 13 个 | 4:1~6:1 极端 Banner | 走确定性 cover，参考图不影响质量 |

**三个关键比例族**（竖版/方形/横版）覆盖 116/139 = **84%** 的可产出尺寸。若用户只上传了横版截图，竖版和方形的20+37=57个尺寸的适配质量会明显下降。

## 目标

1. **告知用户最佳参考图数量**：3张（竖版+方形+横版各一张），覆盖3大比例族
2. **在参考图行展示比例族引导图章（stamp）**：对每个关键比例族渲染一个可视化状态卡，显示该族是否已被已选参考图覆盖
3. **满足最佳时给出正向标记**：当3大比例族全部覆盖时，在参考图区域头部展示"覆盖最佳"徽章

## 方案概述

### 比例族覆盖检测（纯前端）

对每张已选参考图，取其 `width/height` 计算比例 `r`：
- `r < 0.80` → 覆盖**竖版族**
- `0.80 ≤ r ≤ 1.25` → 覆盖**方形族**
- `r > 1.25` → 覆盖**横版族**

一张图只要在该比例范围内即标记对应族已覆盖（无需精确到 9:16）。

### 引导图章展示

在参考图行下方（或右侧）渲染3个「比例族图章」：

```
[竖版 9:16]  [方形 1:1]  [横版 16:9]
   ✓ 已覆盖    · 未覆盖    ✓ 已覆盖
```

每个图章：
- 以对应宽高比绘制一个空白框（线框占位）
- 框内标注比例名（如「竖版」）和代表比例（如「9:16」）
- 覆盖时：边框变为 accent 色 + 小对勾角标，轻微背景填充
- 未覆盖时：虚线边框 + 灰色，暗示"缺这个"

### 最佳标记

参考图行头部：
- 未满足：`已选 N 张` + 灰色小文字 `建议 3 族全覆盖`
- 全覆盖：`已选 N 张` + 绿色徽章 `覆盖最佳 ✓`（accent 色）

## 不做的事

- 不修改后端或 API
- 不在图章上做交互式上传（用户仍通过对话上传图片）
- 不强制要求3张，仍允许1张参考图启动集邮册
- 超宽族（>2.6）不列为引导族，因其走确定性 cover

## 影响范围

仅 `web/src/components/workspace/stamp-album.tsx`，纯前端展示逻辑。
