# Change Proposal: stamp-album-workspace

## Summary
将工作区的「批量切尺寸」工作流替换为「集邮册」视图模式——一个以参考图为锚点、按渠道排布所有尺寸插槽的宣发素材全览面板。用户选定平台后可一键触发全渠道适配，生成的素材自动回填到对应插槽，形成可直接交付的「集邮册」。

## Motivation
当前的批量切尺寸入口是工具栏 → 对话框（SizePicker）的两步流程，用户需要手动勾选尺寸，缺乏对"这个平台还差哪些图"的全局感知。宣发团队实际工作中需要的是：看到整张「清单」，逐渠道批量生成，确认所有格子都被填满后打包下载。

## Changes

### 删除
- `WorkspacePanel` 工具栏中的「批量切尺寸」入口按钮

### 新增：集邮册视图
- 工作区新增第三种视图模式 `stamp`（与现有 `grid` / `timeline` 并列，图标：`BookImage`）
- 视图分两区：
  - **顶部 — 参考图区**：展示工作区内用户指定的参考图（即 `referenceAsset`，缺省时可选任意已有图），附「更换」按钮；参考图用于后续一键生成
  - **底部 — 集邮册网格**：按 `channels.json` 渲染全部渠道，每个渠道为一列/一组，渠道内每个可生产尺寸（`producible: true`）为一个插槽
- 每个插槽展示：
  - 空态：尺寸名、宽×高，视觉上类似空邮票框（虚线边框）
  - 填充态：生成图缩略图，鼠标悬停可预览/下载/重新生成
  - 进行中态：与现有 TaskCard 对齐的骨架 + 进度

### 自动回填
- 当任意 asset 落库时（`asset_ready` 事件），若其 `sizeId` 能对应集邮册中某个插槽，该插槽自动更新为已填充态；无需用户手动刷新

### 一键生成
- 每个渠道组右上角提供「生成全部」按钮，点击后向 Agent 发送适配指令（走现有 `platform-adaptation` 路径），为该渠道下所有未填充、可生产的尺寸各产出一张

## Out of Scope
- 单个资产卡片上现有的「切尺寸」入口（保留，用于细粒度调整）
- SizePicker 对话框本身（保留，仍被单图切尺寸入口调用）
- 非 `producible` 规格（视频等）不在集邮册生成范围内，仅展示占位说明

## Spec Deltas
1. **stamp-album**（新 spec）：集邮册视图的布局、插槽状态机、自动回填、一键生成
2. **asset-workspace**（delta）：REMOVED 批量切尺寸工具栏入口；MODIFIED 视图模式枚举增加 `stamp`

## Risks & Open Questions
- **参考图选取**：首次进入集邮册时无参考图的 fallback 策略（建议：显示「请上传或生成一张参考图」引导）
- **插槽对应关系**：`sizeId` 已在 `Asset` 类型中存在，可直接用于匹配；但同一 sizeId 可能多次生成（用户重做），建议集邮册展示最新一张
- **渠道太多（27个）**：建议默认折叠未展开的渠道组，或按 group（外渠/手机厂商/腾讯内渠/PC）分 tab 过滤
