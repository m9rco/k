## ADDED Requirements

### Requirement: 图生 Icon
系统 SHALL 支持用户基于工作区中的某张图，请求生成与该图相关的 icon。此为**调用图生图大模型的再创作**，而非纯裁剪。系统 SHALL 接受可选的目标尺寸；当用户未指定尺寸时，默认产出 **150×150**。生成 SHALL 复用既有异步任务/进度反馈/工作区回填管线。用户文本经注入防护后由服务端 icon 模板组装最终提示。

#### Scenario: 按图生成 icon（未指定尺寸）
- **WHEN** 用户选择某张图并要求"生成相关 icon"且未给出尺寸
- **THEN** 系统以该图为参考调用图生图大模型生成 icon
- **AND** 最终产物尺寸为 150×150
- **AND** 产物作为图片资产回填工作区，进度经实时通道反馈

#### Scenario: 按图生成 icon（指定尺寸）
- **WHEN** 用户要求生成 icon 并指定了尺寸（如 512×512）
- **THEN** 系统产出该指定尺寸的 icon

#### Scenario: 尺寸收敛
- **WHEN** 图生图供应商仅支持固定尺寸枚举、不能直接产出目标 icon 尺寸
- **THEN** 系统以供应商合法尺寸出图后，由纯图像处理收敛到目标 icon 尺寸（默认以不裁主体的方式）
- **AND** 用户视角为一次性产出目标尺寸的 icon

#### Scenario: icon 提示模板与注入防护
- **WHEN** 用户在描述 icon 时包含试图改写系统指令的内容
- **THEN** 系统通过结构化 slot 承接用户输入并由服务端 icon 模板组装最终提示
- **AND** 模板强调从源图提炼主体、构图居中留边、适配小尺寸辨识度
- **AND** 用户文本不被直接拼接为可改写系统行为的提示
