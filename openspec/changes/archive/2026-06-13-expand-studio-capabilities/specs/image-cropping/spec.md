## ADDED Requirements

### Requirement: 裁剪直连端点
系统 SHALL 提供一个直连的裁剪端点（不经会话 agent/LLM），接受源资产 id 与一组尺寸 id，执行纯图像裁剪并将产物作为资产回填工作区，同步返回每个产物的 id、宽高与渠道/尺寸归属。裁剪是确定性的纯图像处理，因此 SHALL NOT 依赖模型推理；agent 的对话式裁剪工具继续保留用于自然语言请求。

#### Scenario: 直连裁剪并回填
- **WHEN** 前端对某个源资产提交一组尺寸 id 到裁剪端点
- **THEN** 系统执行纯图像裁剪并将每个产物持久化为工作区资产
- **AND** 返回各产物的 id、宽高与渠道/尺寸归属，不经过 agent/LLM

#### Scenario: 无效或不可裁剪 id 报错
- **WHEN** 提交的尺寸 id 不存在或标记为不可裁剪产出
- **THEN** 端点返回明确错误，不产出该尺寸的图片

### Requirement: 裁剪产物分发策略
当一次裁剪的产出文件数在阈值（6）以内时，系统（前端）SHALL 将产物直接呈现在工作区；当产出文件数超过该阈值时，系统 SHALL 改为将产物按渠道/尺寸分目录打包为 zip 供下载，并在执行前向用户给出前置提示。产出文件数按「所选尺寸数 × 源资产数」计。

#### Scenario: 产物数在阈值内直接回填
- **WHEN** 一次裁剪产出的文件数不超过 6
- **THEN** 产物直接作为新资产呈现在工作区

#### Scenario: 产物数超阈值打包下载
- **WHEN** 一次裁剪产出的文件数超过 6
- **THEN** 前端在执行前提示将打包下载
- **AND** 完成后将产物按渠道/尺寸分目录打包为 zip 供用户下载
