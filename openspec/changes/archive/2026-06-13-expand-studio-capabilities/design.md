## Context
这批需求新增三类外部能力（爬取、图生视频、图片优化），扩展生图入参，并补充工作区/上下文的清理操作。多数改动彼此独立，可并行落地。爬取与图生视频引入新的外部依赖与长任务形态，需复用既有的异步任务 + SSE 进度框架。删除类操作（清理/移除）不可逆，需明确确认与会话作用域隔离。

## Goals / Non-Goals
- Goals:
  - 把素材来源（爬取）与图→视频产出补进主链路，复用现有 task/SSE/工作区回填机制。
  - 生图支持最多 6 张参考图。
  - 工作区/上下文可清理，单图可移除。
  - 图片产物默认 PNG 无损优化，可关闭。
  - 下一步提示走前端启发式，零后端成本。
- Non-Goals:
  - 不做爬取结果的商用再分发或版权处理（仅预览信息获取）。
  - 不做视频的多参考/首尾帧等高级控制（单图 + 动作词）。
  - 不引入 JPEG 有损/近无损重编码（仅 PNG 真无损）。
  - 预测不调用 LLM（不做服务端意图预测）。

## Decisions

### D1: 多参考图（需求2）
- **决策**：生图入参由单 `SourceAssetID` 扩展为 `ReferenceAssetIDs []string`（≤6，超出截断并提示）；`Request` 增加 `ReferenceImages [][]byte`。主参考图（第一张）继续驱动调色板与尺寸继承；其余作为附加参考传给 provider（OpenAI 兼容 images/edits 的多图入参，provider 不支持时退化为仅用主图 + 在 prompt 中描述）。
- **理由**：复用构图/风格/角色常需多图。保持向后兼容：单图调用 = 长度 1 的列表。
- **取舍**：gpt-image 系列对多图支持不一，采用「能传则传、不能则主图 + prompt 描述」的务实降级。

### D2: 图生视频（需求3）
- **决策**：新增 `internal/video` 服务，复用 generation 的「异步 task + SSE 进度 + 资产回填」骨架；provider 目标为预留的 `happyhorse-1.0-r2v`（yunwu/alibailian 端点）。入参：源图 + 动作描述（经注入防护的结构化 slot）。产物为视频资产（新 asset kind `video`，mime `video/mp4`）。新增 agent 工具 `image_to_video`。
- **理由**：.env 已预留该 provider，r2v = reference-to-video，正好匹配「单图 + 动作」。
- **风险**：provider 未实跑过；服务层做成可降级（provider 未配置则该意图返回「未配置」而非崩溃）。

### D3: 游戏素材爬取（需求1）
- **决策**：新增 `internal/crawl` 服务，按游戏名搜索素材源、抓取图片预览，作为 `upload`/`crawled` kind 资产回填工作区。复用异步 task + SSE。新增 agent 工具 `crawl_game_assets`。爬取仅取预览图，标注来源，受 robots/速率约束，失败按条跳过。
- **理由**：用户选择按游戏名搜索抓取。
- **风险/取舍**：外部源不稳定、可能无结果——spec 要求「无结果时明确反馈、按条跳过失败」，不静默成功。具体搜索源在实现阶段确定（design Open Question）。

### D4: 图片无损优化（需求9）
- **决策**：新增 `internal/imageopt`，对 PNG 做真无损优化（重新编码 + 去除非必要块，像素一致）。生图/裁剪产物落库前、下载/打包时按开关应用；默认开启。每个涉及图片产出的工具/操作携带 `lossless` 布尔（默认 true）。非 PNG 原样保留。
- **理由**：用户要求默认程序侧无损压缩。PNG 无损可严格保证像素不变。
- **取舍**：不碰 JPEG（无法真无损显著压缩）；优化失败时回退原图，不阻断产出。

### D5: 工作区/上下文清理与单图移除（需求4/5/6）
- **决策**：
  - 工作区清理：新增端点删除会话全部资产（及文件）与未完成任务占位。
  - 单图移除：新增端点删除单个资产（含文件），从工作区移除。
  - 上下文清理：`Orchestrator` 提供重置会话 Window 的方法（保留 system 提示），经 WS/HTTP 触发。
- **理由**：长会话需要主动收拾现场、开新话题。
- **安全**：删除是高影响操作，前端需二次确认（清空工作区/清上下文）；后端按会话作用域，不跨会话。

### D6: 下一步预测与自动补全（需求7）
- **决策**：纯前端启发式：依据本地历史动作序列、偏好关键词（已有）、当前工作区状态（有无选中、最近产物 kind），给出 1–3 条补全提示（如刚生图→提示「切尺寸 / 加动效 / 下载」）。无记录时不展示。
- **理由**：用户选择前端启发式，零后端、即时。

## Risks / Trade-offs
- **视频/爬取 provider 不稳定或未配置** → 服务层可降级：未配置则礼貌告知「未配置/暂不可用」，不崩溃。
- **删除操作不可逆** → 前端二次确认 + 后端会话作用域隔离；spec 写明确认要求。
- **多图 provider 兼容性** → 主图 + prompt 描述的降级路径。

## Migration Plan
1. 先上低风险独立项：图片优化（D4）、工作区/上下文清理与移除（D5）、前端自动补全（D6）。
2. 多参考图（D1）：扩展入参，保持单图兼容。
3. 新外部能力（D2 视频、D3 爬取）：各自带可降级路径，provider 未配置时不崩溃。
4. 全程 `go test ./...` + 浏览器手测；爬取/视频在配齐 provider 的环境单独验证。
