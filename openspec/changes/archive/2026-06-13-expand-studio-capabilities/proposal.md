# Change: 核心能力扩展（爬取 / 图生视频 / 多参考图 / 清理与压缩）

## Why
体验优化告一段落后，进入能力补全阶段。当前 Agent 只能在「单图编辑 + 裁剪 + 下载」闭环内工作：缺少素材来源（爬取）、缺少图→视频这一关键产出形态、生图只能用单张参考、工作区与上下文无法清理、图片产物未做体积优化。这批需求把工作台从「改图工具」补全为「素材生产 + 管理」的完整链路。

## What Changes
- **游戏素材爬取（需求1）**：会话中按**游戏名**发起素材爬取，抓取该游戏的图片素材预览回填工作区（仅信息获取、非商用再分发）。激活 conversation-orchestration 白名单中预留的「物料爬取」意图。
- **多参考图生图（需求2）**：生图/二次调整支持**最多 6 张**参考图（而非当前单张），用于复用构图、风格、角色等。**BREAKING**（生图入参由单 `SourceAssetID` 扩展为参考图 id 列表）。
- **图生视频（需求3）**：基于**单张图 + 动作描述**（如「让角色走起来」），经预留的 happyhorse-1.0-r2v 供应商产出短视频，回填工作区。激活白名单中预留的「生视频」意图。
- **工作区清理（需求4）**：一键清空当前会话工作区的全部资产与任务占位。
- **上下文清理（需求5）**：一键重置会话的 context 滑动窗口（保留 system 提示），让用户主动开新话题。
- **移除单张工作区图片（需求6）**：对工作区单个资产做移除操作。
- **下一步预测与自动补全提示（需求7）**：前端基于本地历史/偏好关键词与当前工作区状态，做启发式的下一步补全提示（零后端）。
- **图片无损压缩（需求9）**：所有图片产物（生图/裁剪/下载）默认经程序侧 **PNG 无损优化**（重压缩、去冗余块，像素不变），并提供可关闭的开关。
- **裁剪流程改造（追加）**：裁剪改为前端**直连 HTTP 端点**（纯图像处理，不经 agent/LLM），产物即时回填工作区；产出文件数 >6 时改为按渠道/尺寸**分目录打包下载**并前置提示。资产卡展示尺寸标识。工作区任务按状态阶段**里程碑式分组**（进行中/已完成/失败）。

## Impact
- Affected specs:
  - 新增：`material-crawling`、`video-generation`、`image-optimization`
  - 修改：`image-generation`（多参考图）、`asset-workspace`（清理/移除）、`conversation-orchestration`（上下文清理、白名单激活）、`frontend-experience`（自动补全提示、清理/移除入口、压缩开关、视频预览、尺寸标识、里程碑分组）、`image-cropping`（直连端点、产物分发策略）
- Affected code:
  - 后端: 新增 `internal/crawl`、`internal/video`、`internal/imageopt`；`internal/generation`（多参考图）、`internal/agent`（新工具 + prompt 白名单）、`internal/workspace`（清理/移除端点）、`internal/config`（视频/爬取配置）、`internal/store`（资产删除）
  - 前端: `web/static/app.js`、`index.html`、`styles.css`（自动补全、工作区/上下文清理、单图移除、参考图多选、压缩开关、视频卡片播放）
