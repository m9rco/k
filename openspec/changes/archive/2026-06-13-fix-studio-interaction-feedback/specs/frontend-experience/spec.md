# frontend-experience delta

## MODIFIED Requirements

### Requirement: Agent 式对话与工具调用呈现
系统 SHALL 提供参考主流 Agent 产品（如 Cursor）的对话区，将对话区作为产品核心区域；工具调用 SHALL 以结构化卡片呈现，并严格受控地展示每次调用的生命周期状态（待执行/执行中/成功/失败）与结果摘要，同时清晰呈现 context 状态与当前会话信息。工具调用卡片 SHALL 以**图标 + 简短中文意图短语**为主标题（如「🎨 换背景」「🎬 生成视频」「✂️ 切尺寸」），并以人类可读的关键参数（如背景描述、动作描述、尺寸数量）作为副信息，**不得**将原始参数 JSON 作为卡片主体展示。

#### Scenario: 展示工具调用生命周期
- **WHEN** Agent 调用某个工具（生图/裁剪/下载等）
- **THEN** 对话区以结构化卡片展示该工具名与参数摘要
- **AND** 卡片随执行进展严格更新为执行中→成功/失败，并在结束时展示结果摘要或错误
- **AND** 任一工具调用不会停留在无结局的"执行中"状态

#### Scenario: 工具卡片人性化呈现
- **WHEN** Agent 调用 `edit_image`（如 change_background）等工具
- **THEN** 卡片主标题以图标加中文意图短语呈现（如「🎨 换背景」），副信息展示关键参数的可读摘要（如「淡紫色背景」）
- **AND** 卡片不把 `{"intent":...,"source_asset_id":...}` 之类原始 JSON 作为主体内容直接呈现给用户

#### Scenario: Context 状态面板
- **WHEN** 会话进行中
- **THEN** 界面展示当前 context 使用状态与当前会话信息

### Requirement: 思考过程流式展示与折叠
系统 SHALL 将 Agent 的思考过程以**逐字流式（打字机式）**方式实时展示在对话区，作为与最终回答视觉区分的"思考块"；思考文本 SHALL 在到达时按固定/自适应速率逐字涌现，使用户在等待期间即可看到思考在进行，而非一次性整段出现或长时间空白等待；当本轮回答开始产出或进入工具调用时，思考块 SHALL 自动折叠为收起态，并保留可展开入口供用户回看。

#### Scenario: 思考逐字涌现
- **WHEN** Agent 在产出回答前进行思考且模型返回思考内容
- **THEN** 前端以打字机式逐字增量方式实时渲染思考文本于独立的思考块
- **AND** 思考块在视觉上与最终回答明显区分
- **AND** 用户在思考进行期间持续看到文字逐步涌现，不会面对长时间无变化的等待

#### Scenario: 产出结论后折叠
- **WHEN** 本轮回答开始产出或工具调用开始
- **THEN** 系统先将思考块中尚未吐完的字符补完，再将其折叠为收起态
- **AND** 用户可点击展开回看完整思考内容

#### Scenario: 无思考内容时不展示
- **WHEN** 模型未返回任何思考内容
- **THEN** 前端不展示空的思考块

## ADDED Requirements

### Requirement: 长任务即时占位反馈
当系统创建长任务（生图、二次调整、生视频等）时，系统 SHALL 在任务创建的瞬间经对话通道广播任务创建通知，前端据此**立即**于工作区「进行中」分组插入占位骨架卡片，并即时订阅该任务的进度流，使工作区在本轮对话尚未结束前即给出可见的 loading 反馈，而非等到对话回答完成才刷新。该占位机制 SHALL 独立于 Agent 的工具回调时序，确保任务一经创建占位即出现。

#### Scenario: 二次调整即时出现占位
- **WHEN** 用户在工作区点击某张图发起二次调整，且系统据此创建了生图任务
- **THEN** 工作区在任务创建后随即在「进行中」分组出现一张占位骨架卡片并展示阶段化 loading 进度
- **AND** 该占位在本轮对话回答结束之前即已出现，无需等待对话 done

#### Scenario: 占位随进度演进至完成
- **WHEN** 该任务的进度事件持续到达
- **THEN** 占位骨架按阶段更新进度，任务完成后替换为最终产物卡片
- **AND** 同一任务 id 在工作区不重复出现多张卡片

#### Scenario: 占位不依赖工具回调时序
- **WHEN** Agent 的工具完成回调延迟、缺失或时序不确定
- **THEN** 占位骨架仍在任务创建时即出现，不受该回调影响

### Requirement: 多图参考改图入口
系统 SHALL 为"用多张已选素材作为参考改另一张图"提供明确可见的入口与状态提示：当用户在工作区多选资产时，界面 SHALL 显式提示已选作为参考的数量并提供一键清除；用户从放大/二次调整面板发起改图时，系统 SHALL 支持把当前已选的多张资产一并作为参考（第一张为主参考）发起，而非只能使用单张。

#### Scenario: 多选时显示参考态
- **WHEN** 用户在工作区选中一张或多张资产
- **THEN** 界面在输入区附近显示「参考 N 张」之类的状态标识并提供清除入口
- **AND** 选中 2 张及以上时文案明确表达这些图将作为参考用于改图

#### Scenario: 从二次调整发起多图参考
- **WHEN** 用户已多选若干资产并从放大/二次调整面板发起改图
- **THEN** 系统将这些已选资产作为参考集（第一张为主参考）发起生图
- **AND** 仅选中单张时维持以该单张为源图的原有行为

#### Scenario: 清除参考选择
- **WHEN** 用户点击参考态标识的清除入口
- **THEN** 已选参考被清空，状态标识消失

### Requirement: 图生视频显式入口
系统 SHALL 在工作区为单张图片提供显式的「生成视频」入口，使用户无需依赖特定对话话术即可发起图生视频：在放大/二次调整面板中 SHALL 提供独立的动作描述输入框与「生成视频」按钮，并在资产右键菜单中 SHALL 提供「生成视频」项。触发时系统以该图为源、用户的动作描述发起图生视频；当动作描述为空时 SHALL 提示用户先描述动作，当视频供应商未配置时 SHALL 礼貌告知而不崩溃。

#### Scenario: 从放大面板生成视频
- **WHEN** 用户放大一张图、在生成视频输入框填写动作描述并点击「生成视频」
- **THEN** 系统以该图为源、动作描述发起图生视频任务
- **AND** 工作区即时出现该视频任务的占位与进度反馈

#### Scenario: 右键菜单进入生成视频
- **WHEN** 用户在某张图上右键并选择「生成视频」
- **THEN** 系统打开放大面板并聚焦到生成视频的动作描述输入框

#### Scenario: 动作描述为空时提示
- **WHEN** 用户点击「生成视频」但未填写动作描述
- **THEN** 界面提示用户先描述想要的动作，且不发起任务
