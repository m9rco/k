# asset-workspace — Delta

## ADDED Requirements

### Requirement: 资产显示编号
系统 SHALL 为工作区中的每个资产分配一个**按当前显示顺序**递增的编号（图1、图2、……），编号从 1 开始连续。当资产被移除或被拖拽重排时，编号 SHALL 实时重新计算以始终反映当前可见顺序。该编号 SHALL 在前端资产卡片上以角标形式展示，使用户可用"图N"指代具体资产。编号 SHALL 为派生展示属性，不持久化到存储。

#### Scenario: 按显示顺序编号
- **WHEN** 工作区按当前顺序展示若干资产
- **THEN** 第 1 个资产标为"图1"、第 2 个为"图2"，以此类推，连续且从 1 开始

#### Scenario: 移除后重编号
- **WHEN** 用户移除工作区中的某个资产
- **THEN** 其后资产的编号实时前移，保持连续

#### Scenario: 重排后重编号
- **WHEN** 用户拖拽重排资产顺序
- **THEN** 编号随新顺序实时更新，始终反映当前可见位置

### Requirement: 资产编号用于与 LLM 沟通
系统 SHALL 使 agent 能够基于资产编号与用户双向沟通：当用户消息中以"图N"指代资产时，系统 SHALL 向 agent 提供"编号 → asset_id"的映射上下文，使 agent 能把"图N"解析为对应资产并填入工具参数；agent 在面向用户的说明性文字中亦 SHALL 可以"图N"称呼资产。编号映射 SHALL 反映用户当前的显示顺序（含用户拖拽重排的结果）。

#### Scenario: 用户以"图N"指代被正确解析
- **WHEN** 用户发送"根据图2和图3生成一张新图"且工作区已有对应资产
- **THEN** 系统向 agent 提供编号映射，agent 将"图2""图3"解析为对应 asset_id

#### Scenario: 编号映射反映重排顺序
- **WHEN** 用户拖拽重排资产后以"图N"指代
- **THEN** 提供给 agent 的编号映射反映重排后的顺序，"图N"对应重排后的第 N 个资产

#### Scenario: agent 以"图N"回应
- **WHEN** agent 在回复中需要指代某资产
- **THEN** agent 可使用"图N"称呼，与用户看到的卡片角标一致
