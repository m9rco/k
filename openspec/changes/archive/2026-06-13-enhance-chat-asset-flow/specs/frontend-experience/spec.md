# frontend-experience — Delta

## MODIFIED Requirements

### Requirement: 思考过程流式展示与折叠
系统 SHALL 将 Agent 的思考过程以**逐字流式（打字机式）**方式实时展示在对话区，作为与最终回答视觉区分的"思考块"；思考文本 SHALL 在到达时按固定/自适应速率逐字涌现，使用户在等待期间即可看到思考在进行，而非一次性整段出现或长时间空白等待；当本轮回答开始产出或进入工具调用时，思考块 SHALL 自动折叠为收起态，并保留可展开入口供用户回看。当一轮结束且既无回复正文、又无工具调用、又无反问产出时，前端 SHALL NOT 渲染空白的回复气泡；若该轮确无任何可呈现内容，前端 SHALL 以一条保底说明替代，避免出现"思考完却无回复"的断裂。

#### Scenario: 思考逐字涌现
- **WHEN** Agent 在产出回答前进行思考且模型返回思考内容
- **THEN** 前端以打字机式逐字增量方式实时渲染思考文本于独立的思考块
- **AND** 思考块在视觉上与最终回答明显区分
- **AND** 用户在思考进行期间持续看到文字逐步涌现，不会面对长时间无变化的等待

#### Scenario: 产出结论后折叠
- **WHEN** 本轮回答开始产出或工具调用开始
- **THEN** 系统先将思考块中尚未吐完的字符补完，再将其折叠为收起态
- **AND** 用户可点击展开回看完整思考内容

#### Scenario: 无思考内容时不展示
- **WHEN** 模型未返回任何思考内容
- **THEN** 前端不展示空的思考块

#### Scenario: 空回复不留空气泡
- **WHEN** 一轮结束且无回复正文、无工具调用、无反问
- **THEN** 前端不渲染空白回复气泡
- **AND** 若该轮确无任何可呈现内容，前端以一条保底说明替代

## ADDED Requirements

### Requirement: 回复 Markdown 渲染
前端 SHALL 对 agent 回复正文中的 markdown 标记进行**轻量安全渲染**：支持加粗、斜体、标题、有序/无序列表、链接、行内代码；代码块以纯文本预格式块展示其内容；SHALL NOT 渲染原始 HTML（防注入），链接 SHALL 仅允许 http(s) 协议。渲染 SHALL 容忍流式过程中的未闭合标记（半截 markdown 不破坏展示）。前端 SHALL NOT 将 markdown 源码原样暴露给用户，也 SHALL NOT 简单剥离标记导致内容残缺。

#### Scenario: 常见 markdown 被渲染
- **WHEN** agent 回复包含加粗、列表、标题、链接、行内代码等 markdown
- **THEN** 前端渲染为对应富文本，不显示 `**`、`#`、`-` 等源码标记

#### Scenario: 代码块按纯文本块展示
- **WHEN** agent 回复包含围栏代码块
- **THEN** 前端以纯文本预格式块展示其内容，不作为可执行/可注入内容

#### Scenario: 禁止原始 HTML 注入
- **WHEN** agent 回复中含原始 HTML 标记
- **THEN** 前端不渲染该 HTML，按安全文本处理

#### Scenario: 流式半截标记不破坏渲染
- **WHEN** 流式过程中出现未闭合的 markdown 标记（如尚未配对的 `**`）
- **THEN** 前端按纯文本容错处理，不破坏整体渲染

### Requirement: 资产编号角标
前端 SHALL 在每个工作区资产卡片上展示其**按显示顺序**的编号角标（图1、图2、……），编号随移除/重排实时更新，使用户可用"图N"指代资产。该角标 SHALL 与既有的类型标签、尺寸标识共存且不互相遮挡。

#### Scenario: 卡片展示编号角标
- **WHEN** 工作区按顺序展示资产
- **THEN** 每张卡片展示其当前序号的"图N"角标

#### Scenario: 角标随顺序更新
- **WHEN** 用户移除或重排资产
- **THEN** 各卡片的"图N"角标实时更新以反映当前显示顺序

### Requirement: 消息队列与插队
前端 SHALL 在会话进行中（一轮处理未结束）将用户的新输入加入**待发队列**而非立即并发发送，并在界面以可见方式展示队列中的待发消息。前端 SHALL 在当前轮结束后按队列顺序自动出队发送。前端 SHALL 提供两种插队方式：将队列中某条消息**手动提前**到队首，以及**打断当前轮**（发送中断信号并立即发送指定消息）。

#### Scenario: 进行中输入入队展示
- **WHEN** 一轮处理进行中，用户提交新输入
- **THEN** 前端将其加入待发队列并在界面展示，而非立即发送

#### Scenario: 轮结束自动出队
- **WHEN** 当前轮结束且队列非空
- **THEN** 前端自动出队队首消息并发送

#### Scenario: 手动提前
- **WHEN** 用户将队列中某条消息移到队首
- **THEN** 该消息成为下一个发送的消息

#### Scenario: 打断当前轮立即发送
- **WHEN** 用户选择打断当前轮并发送某条消息
- **THEN** 前端发送中断信号后立即发送该消息
- **AND** 界面反映当前轮被打断、新消息开始处理
