# Change: 强化 gpt-image-2 多参考图低发散适配 harness

## Why

当前生图 harness 是按 `gpt-image-1` 的事实建模的，与已切默认的 `gpt-image-2` 存在三处**事实性偏差**，直接损害「给 1~16 张参考图、产出构图和谐且不发散的各目标尺寸图」这一核心诉求：

1. **尺寸枚举写死成 3 档**。`legalGenSizes` 硬编码 `1024×1024 / 1536×1024 / 1024×1536`（这是 gpt-image-1 的固定档），而 gpt-image-2 接受**近任意分辨率**（约束：最长边 ≤3840、长短边均为 16 倍数、长短比 ≤3:1、总像素 ∈ [655360, 8294400]，>2K 实验性）。结果：1080×1920 这类目标被强行就近到 2:3，再由收敛步骤裁/补——**白白引入一次形变**，构图和谐度下降，且无法利用 2048/4K 等原生档。
2. **参考图上限 6 张且无角色分层**。spec/代码均为「第一张主参考 + 其余附加」的扁平模型，上限 6。用户要 1~16 张，且多图最大的坑就是**主体漂移 / 语义发散**：模型把多张图"平均"成一个新主体。缺少「锚点图 = 内容与意图的唯一真相源，其余仅供风格/元素参考」的强约束。
3. **保真与平台坑未建模**。gpt-image-2 **自动按高保真处理每张输入图**（不可传 `input_fidelity`），这恰是「不发散」的天然抓手，却未在 harness 利用/记录；同时 gpt-image-2 **不支持透明底**，而尺寸目录存在「透明底」语义约束 —— 当前会把无法满足的约束注入 prompt，诱发模型自由发挥。

本 change 把 harness 从「gpt-image-1 假设」迁到「gpt-image-2 事实」，并把「**和谐优先、保留核心意图、抑制幻觉**」从隐式期望升级为**结构化、可测**的 prompt 与尺寸契约。

## What Changes

> 已核实 `configs/channels.json` 全部 **139 个 producible 档位** vs gpt-image-2 约束：**57%（79 个）低于出图下限 655360px**、11 个长短比 >3:1、2 个 >2K、2 个透明底。**用户硬约束：产物尺寸必须精确等于这些档位，绝不偏离**；gpt-image-2 出图尺寸只是中间值，一律收敛到目录精确像素。

- **按适配器隔离的尺寸解析（gpt-image-2 近任意分辨率，恒收敛到目录精确档位）**：新增 gpt-image-2 专属尺寸解析器，在其官方约束内贴合目标宽高比出图；因半数以上档位小于出图下限，**主路径是「按目标比例放大到 ~2K 出图 → 等比下采样到目录精确尺寸」**（下采样使小图更锐利）；极端比例（>3:1）夹到 3:1 出图再 `cover` 裁到精确档位。DashScope 等固定枚举供应商保留各自映射，互不影响。**BREAKING**（spec 层）：废止「gpt-image-2 合法枚举为 1024/1536 三档」的旧需求表述。
- **参考图上限 1→16 且引入「锚点 + 辅助」角色分层**：`MaxReferenceImages` 6→16；第一张为**锚点图**（内容、主体身份、核心宣发意图的唯一真相源），其余为**辅助参考**（仅贡献风格/元素/构图灵感，不得改写锚点主体）。prompt 模板显式声明该角色契约。
- **低发散 prompt harness（上下文 + 保留-修改-禁止 四段式）**：所有图生图意图的服务端模板统一加入 `[CONTEXT]`（声明「现有游戏的宣发素材、参考图为该游戏真实美术、停留在该游戏既有风格世界观内」）+ `[PRESERVE]`/`[MODIFY]`/`[AVOID]` 三段，把「贴合游戏宣发、和谐、保留意图、不发散、**不虚构游戏中不存在的画面**」编码为模型可执行指令。`[CONTEXT]`/`[PRESERVE]`/`[AVOID]` 为服务端固定文案，杜绝注入。
- **保真与平台坑建模**：harness 记录「gpt-image-2 自动高保真、不传 input_fidelity」事实（这是"不发散"的天然抓手）；对 gpt-image-2 **不支持透明底**的 2 个档位做能力探测——不可满足约束改写为「中性底便于抠图」而非诱发模型瞎画。
- **iOS 2732×2048 等超 2K 档位**默认夹 2K 出图后由收敛上采样，不默认放行实验档。

## Impact

- Affected specs: `image-generation`（参考图复用、低发散提示骨架[新增]、保真与能力坑建模[新增]）、`platform-adaptation`（生图尺寸就近映射、适配尺寸精确收敛）
- Affected code:
  - `internal/generation/http_provider.go`（`legalGenSizes` / `sizeParam` / `nearestGenSize` → gpt-image-2 尺寸解析器）
  - `internal/generation/prompt.go`（四段式低发散模板、游戏宣发 `[CONTEXT]`、锚点角色契约、透明底等约束改写）
  - `internal/generation/service.go`（`MaxReferenceImages` 6→16、锚点/辅助分层透传）
  - `internal/generation/provider.go` / `dashscope.go` / `gemini.go`（尺寸解析按适配器隔离的边界确认）
  - `internal/generation/adapt.go`（收敛与新尺寸解析的协同：恒收敛到目录精确档位）
  - 相应 `_test.go`（表驱动覆盖新尺寸解析、角色分层、四段式模板）
